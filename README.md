## рҹ”„ Flujo del Wizard

El sistema guГӯa al operador mediante un **wizard de 5 pasos** con transiciones animadas:

| Paso | Vista | DescripciГіn |
|:----:|-------|-------------|
| **1** | рҹ“ё **Captura** | DigitalizaciГіn fГӯsica mediante NAPS2 o carga de PDF existente. Opciones de duplex y resoluciГіn (150вҖ“600 DPI). |
| **2** | рҹ‘ҒпёҸ **Preview** | Editor interactivo con drag & drop para reordenar, rotar (В°90) o eliminar pГЎginas antes del procesamiento. |
| **3** | рҹӨ– **ExtracciГіn IA** | ClasificaciГіn automГЎtica del documento, extracciГіn de metadatos y validaciГіn de firmas. |
| **4** | вң… **ValidaciГіn** | RevisiГіn humana de campos extraГӯdos: clasificaciГіn, folio, remitente, fecha y asunto. Indicadores de confianza. |
| **5** | рҹҺү **ConfirmaciГіn** | Registro exitoso con folio asignado, subida a Google Drive, notificaciГіn por email y asiento en Libro de Gobierno Digital. |

## вҡҷпёҸ ConfiguraciГіn

### Variables de Entorno

Todas las configuraciones se gestionan mediante variables de entorno con el prefijo `SCANBRIDGE_`:

| Variable | Default | DescripciГіn |
|---|---|---|
| `SCANBRIDGE_RAW_PDF_PATH` | (OS temp dir)`/raw_scan.pdf` | Ruta temporal del PDF escaneado. Usa `tempfile.gettempdir()`. |
| `SCANBRIDGE_PROCESSED_PDF_PATH` | (OS temp dir)`/oficialia_final_processed.pdf` | Ruta del PDF procesado final. Usa `tempfile.gettempdir()`. |
| `SCANBRIDGE_SCANNER_PROFILE` | `Oficialia_Estandar` | Perfil de escГЎner NAPS2 |
| `SCANBRIDGE_DEFAULT_RESOLUTION` | `300` | DPI por defecto para escaneo |
| `SCANBRIDGE_THUMBNAIL_DPI` | `72` | DPI para generaciГіn de miniaturas |
| `SCANBRIDGE_HOST` | `127.0.0.1` | Host del servidor WebSocket |
| `SCANBRIDGE_PORT` | `8000` | Puerto del servidor WebSocket |
| `SCANBRIDGE_SCAN_TIMEOUT` | `300` | Timeout de escaneo en segundos |

### Ejemplo de configuraciГіn en contenedor

export SCANBRIDGE_HOST=0.0.0.0

## рҹ”Ң Protocolo WebSocket

El frontend y backend se comunican mediante un canal WebSocket bidireccional con mensajes JSON.

### Comandos (Frontend вҶ’ Backend)

```jsonc
// Escaneo fГӯsico
{
  "command": "START_SCAN",
  "duplex": true,
  "resolution": 300
}

// Aplicar ediciones (reordenamiento + rotaciГіn)
{
  "command": "APPLY_EDITS",
  "operations": [
    { "source_index": 0, "rotation": 0 },
    { "source_index": 2, "rotation": 90 },
    { "source_index": 1, "rotation": 0 }
  ]
}

// Carga de PDF local (sin escГЎner)
{
  "command": "LOAD_LOCAL_PDF",
  "base64_data": "data:application/pdf;base64,JVBERi0x..."
}
```

### Eventos (Backend вҶ’ Frontend)

| Evento | Payload | DescripciГіn |
|---|---|---|
| `HARDWARE_STATUS` | `{ online, model }` | Estado del escГЎner fГӯsico |
| `SCAN_STARTED` | вҖ” | Escaneo iniciado |
| `scan_status` | `{ progress, message }` | Progreso del escaneo (0вҖ“100) |
| `SCAN_COMPLETED` | `{ output_path }` | PDF generado en disco |
| `THUMBNAILS_READY` | `{ pages[] }` | Miniaturas listas (base64) |
| `EDITS_APPLIED` | `{ output_path }` | PDF procesado guardado |
| `SCAN_ERROR` | `{ message }` | Error en el proceso |

### OrГӯgenes WebSocket Permitidos

Por seguridad, el servidor solo acepta conexiones de orГӯgenes especГӯficos. Estos se configuran mediante la variable de entorno `SCANBRIDGE_ALLOWED_ORIGINS`.

- **Formato**: Una lista de URLs separadas por comas.
- **Default**: `http://localhost:5173,http://127.0.0.1:5173` (servidor de desarrollo de Vite).
- **ProducciГіn**: Debes configurar esta variable para que coincida con el dominio donde se sirve el frontend.

#### Ejemplo de configuraciГіn

```bash
# Permitir solo el dominio de producciГіn y un alias local
export SCANBRIDGE_ALLOWED_ORIGINS="https://oficialia.miinstitucion.gob,http://localhost:8080"
```

## рҹӣЎпёҸ Seguridad

| Vector | MitigaciГіn |
|---|---|
| **XSS (innerHTML)** | `escapeHTML()` en todos los datos dinГЎmicos |
| **XSS (textos)** | `textContent` para mensajes de usuario |
| **InyecciГіn de comandos** | ValidaciГіn Pydantic con discriminadores `Literal` |
| **Path traversal** | Rutas definidas por ENV, no por input del usuario |

| **ReconexiГіn segura** | Teardown completo del socket anterior antes de reconectar |
